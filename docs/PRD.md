# Task Horizon PRD

版本基线：`plugin.json` 2.3.7  
文档目的：基于当前代码实现，整理给前端重建界面时使用的产品需求、功能范围和验收标准。

## 1. 产品概述

Task Horizon 是思源笔记内的专业任务管理插件。它从多个文档、笔记本或自定义文档分组中聚合任务块，并在插件页签、Dock、移动端文档入口和文档内悬浮条中提供统一的任务浏览、编辑、筛选、统计、日历排期、白板编排和 AI 辅助能力。

核心价值：

- 把散落在思源文档中的任务统一汇总，不改变思源原生块数据模型。
- 提供多种工作视图：主页总览、清单、表格、时间轴、看板、日历、白板。
- 支持任务字段化管理：重要性、状态、开始日期、截止日期、时长、耗时、备注、附件、自定义列、循环规则、提醒。
- 通过文档分组、规则筛选、分组排序和优先级算法，帮助用户从大量任务中定位下一步行动。
- 保持桌面端、Dock、移动端和文档内 quickbar 的一致体验。

## 2. 目标用户

- 思源笔记重度用户：任务散落在多个笔记文档、日记或项目页，需要跨文档聚合。
- GTD / 项目管理用户：希望按文档、标题、时间、状态、重要性、四象限组织任务。
- 移动端随手记录用户：需要在手机上快速新建任务、设置日期和查看今日日程。
- 时间管理用户：使用底栏番茄钟，希望任务耗时、提醒、日历排期联动。
- 高级自定义用户：需要自定义状态、列、主题、筛选规则和优先级算法。

## 3. 使用场景

1. 首次配置任务范围：用户打开插件后，进入设置，添加笔记本分组或自定义文档分组，刷新后看到任务。
2. 日常任务处理：用户在清单视图中查看任务树，点击任务打开详情，修改状态、日期、备注、附件和子任务。
3. 周计划排期：用户切到日历视图或时间轴视图，通过拖拽调整任务日期和日程时间。
4. 项目看板：用户切到看板视图，按状态或标题分组查看任务卡片，并拖拽调整状态或位置。
5. 白板规划：用户在白板中把任务卡片、文档框和便签放在画布上，建立任务关联。
6. 复盘统计：用户进入主页总览，查看完成率、趋势、热力图、文档分布、最近完成和逾期风险。
7. 文档内快速编辑：用户在思源原文档中点击任务块，使用 quickbar 编辑常用字段。

## 4. 范围

### 4.1 本期必须覆盖

- 插件主壳：顶部工具栏、文档分组、文档页签、规则、搜索、刷新、设置、视图切换。
- 主页总览：任务完成统计、趋势、热力图、文档分布、最近完成、逾期风险。
- 任务视图：清单、表格、时间轴、看板、日历、白板。
- 任务详情：标题、位置、完成状态、重要性、状态、开始/截止日期、时长、耗时、提醒、循环、子任务、备注、附件、自定义字段、更多操作。
- 批量操作：清单/表格/时间轴中开启多选后批量修改日期、重要性、状态、自定义字段。
- 设置中心：文档分组、常规、外观、日历、AI、规则、四象限、优先级算法、关于。
- quickbar：文档内任务块悬浮编辑条，以及任务行末尾常驻字段。
- 导入导出：滴答清单 CSV 导入、当前分组任务导出、汇总 Markdown/Excel 类能力。
- 响应式：桌面页签、Dock、移动端、横屏/窄屏的布局适配。

### 4.2 非目标

- 不重新定义思源任务块存储格式。
- 不替代思源原生编辑器；任务内容仍以思源块为最终数据源。
- 不做独立云同步。插件数据保存于工作空间插件存储，可依赖思源/伺服同步文件。
- 不把 AI 作为必需能力；AI 入口需可关闭并完全隐藏。

## 5. 信息模型

### 5.1 任务

任务来自思源任务块，主要字段：

- `id`：任务块 ID。
- `root_id` / `docId`：所属文档。
- `content` / `markdown`：任务标题和原始 Markdown。
- `done`：完成状态，兼容思源任务复选框。
- `parentTaskId` / `children`：父子任务关系。
- `h2` / `h2Id` / `h2Path`：所属标题，标题层级可配置。
- `priority`：重要性，支持高/中/低/无，对应 A/B/C/none。
- `customStatus`：自定义状态。
- `startDate`：开始日期。
- `completionTime`：截止日期。
- `duration`：任务预计时长。
- `spent`：番茄钟耗时。
- `remark`：备注。
- `attachments`：图片、PDF、普通文件、文档链接、块链接。
- `repeatRule`：循环任务规则。
- `pinned`：置顶。
- `milestone`：里程碑。
- 自定义字段：单选、多选、文本列。

### 5.2 文档分组

- 全部文档：聚合所有已选择文档，可维护排除文档列表。
- 笔记本分组：自动搜索笔记本内有任务的文档，分组名称跟随笔记本。
- 自定义分组：手动添加文档 ID，可选择包含子文档。
- 分组内可配置默认新建任务文档、颜色方案、递归文档范围、最近日记文档裁剪。

### 5.3 视图

当前默认视图为清单，支持启用/禁用以下视图：

- `checklist`：清单视图。
- `list`：表格视图。
- `timeline`：时间轴/Gantt 视图。
- `kanban`：看板视图。
- `calendar`：日历视图。
- `whiteboard`：白板视图。

## 6. 功能需求

### 6.1 插件入口

- 桌面端全局顶栏显示任务管理器入口，点击打开插件页签。
- 文档顶栏可显示插件按钮。
- 移动端文档顶栏按钮默认短按快速新建任务，长按打开任务管理器；设置中可对调长短按。
- 开启“打开时定位当前文档”后，从文档顶栏打开插件应优先切到当前文档所在分组和页签。

验收：

- 未配置文档范围时，主界面给出清晰空态并引导进入设置。
- 移动端入口可访问快速新建和完整任务管理器。

### 6.2 主界面壳

主界面由顶部工具栏、文档页签/分组区、内容视图、侧栏/详情区、底部浮动条组成。

顶部工具栏必须包含：

- 主页入口。
- 新建任务。
- 文档分组下拉。
- 当前规则下拉。
- 搜索。
- 分组模式入口：按文档、按时间、按任务名、四象限等。
- 视图切换。
- 刷新。
- 设置。
- 菜单：多选模式等次级功能。

验收：

- 切换分组、规则、视图后应保留可恢复的会话状态。
- 手动刷新可选择是否同步伺服共享设置和会话状态。

### 6.3 主页总览

主页展示当前任务范围内的统计看板：

- 总体完成率、总任务数、未完成数、已完成数、逾期数。
- 完成趋势，支持不同时间范围切换。
- 完成热力图。
- 文档任务分布。
- 最近完成任务。
- 逾期风险任务。

验收：

- 没有任务时显示空态，不出现空图表或报错。
- 点击任务类列表项应能打开任务详情或跳转到任务。

### 6.4 清单视图

清单视图是主工作视图，需支持：

- 任务树缩进、父子任务展开/折叠、完成进度显示。
- 置顶任务分组。
- 按文档、标题、任务名、时间、四象限分组。
- 紧凑模式，右侧显示可配置字段。
- 紧凑层级线，展示子任务层级。
- 有附件任务显示附件标识。
- 点击任务行打开详情；移动端/窄屏使用底部抽屉详情。
- 分组行可折叠，标题分组可直接新建任务。

验收：

- 超过虚拟阈值时应分批/虚拟渲染，滚动不卡顿。
- 完成状态、详情字段修改后清单局部刷新，不应强制整页闪烁。

### 6.5 表格视图

表格视图面向字段密集编辑：

- 列显示/隐藏、顺序、宽度可配置。
- 支持内置列：置顶、任务内容、状态、优先级/得分、文档、标题、重要性、开始日期、截止日期、剩余时间、时长、耗时、备注、附件。
- 支持自定义列。
- 单元格可编辑常用字段。
- 支持横向滚动条和列宽拖拽。

验收：

- 列设置变更后立即反映到表格。
- 附件列可查看/管理附件数量和入口。

### 6.6 时间轴视图

时间轴视图用于计划和排期：

- 左侧表格列：任务内容、开始日期、截止日期。
- 右侧 Gantt 时间轴。
- 支持缩小、放大、适配范围、定位今天。
- 左侧宽度、内容列宽可拖拽调整。
- 侧栏可折叠。
- 支持多选模式。

验收：

- 开始/截止日期编辑后，右侧时间条同步更新。
- 移动端和 Dock 使用浮动工具条，不遮挡多选条。

### 6.7 看板视图

看板视图用于状态流转和标题分组：

- 默认按自定义状态列展示，包含待办和其他状态。
- 标题看板模式可按标题分组，并可显示已完成列。
- 右上角提供新建任务按钮。
- 卡片字段可配置：重要性、状态、日期等。
- 卡片可展示子任务统计、折叠状态、文档信息。
- 支持拖拽调整状态/位置。
- 支持任务详情侧栏。

验收：

- 状态选项变更后看板列同步变化。
- 开启“卡片日期有值才显示”后，无日期任务不显示日期按钮。

### 6.8 日历视图

日历基于 FullCalendar，支持：

- 月、周、日、三日等主视图。
- 日程事件和任务日期事件同时展示。
- 日程新建、编辑、拖拽、调整时间。
- 任务拖拽到日期/时间格以设置截止日期或排期。
- 拖拽任务时可通过浮动重要性图标快速设置重要性。
- 全天区域可折叠，并显示汇总。
- 日历侧边栏：迷你日历、我的日历、文档分组、番茄钟、任务列表。
- 可显示中国节假日、农历。
- 移动端支持长按 1 秒新建日程或拖动已有日程。
- 可与底栏番茄钟联动，显示专注、休息、计时、空闲等事件。
- 可注册移动端原生日程通知。

验收：

- 任务日期变化后，清单详情、日历事件和任务缓存一致。
- 关闭番茄钟联动后，不显示番茄钟相关事件和入口。

### 6.9 白板视图

白板用于任务空间编排：

- 画布模式：平移、缩放、框选、拖拽任务卡片、连接任务、便签。
- 全部页签下支持白板布局和卡片流布局。
- 卡片流按文档组织，可拖拽调整文档卡片顺序。
- 支持按标题子分组。
- 支持显示/隐藏已完成任务。
- 白板任务池可使用不同分组模式。
- 可保存节点位置、文档框尺寸、连接线、便签和视口。
- 移动端可使用任务详情抽屉。

验收：

- 多端同步时白板数据不得因旧版本覆盖新版本。
- 已放置但当前过滤中缺失的任务可通过快照保留必要展示信息。

### 6.10 任务详情

任务详情必须支持：

- 编辑任务标题。
- 显示并切换所属文档、所属标题，提供跳转到任务。
- 更多菜单：删除任务、置顶等任务右键菜单能力。
- 保存、关闭。
- 重要性选择器。
- 状态选择器。
- 开始日期、截止日期、时长、耗时、提醒、循环规则。
- 子任务列表：完成统计、新建子任务、点击子任务进入详情。
- 备注编辑，支持 Markdown 工具。
- 附件区：图片、PDF、普通文件、文档链接、块链接；支持粘贴剪贴板附件或文档 ID。
- 自定义字段：单选、多选、文本。

验收：

- 拖拽鼠标到窗口外时详情页不应误关闭。
- 新建子任务保存过程中，子任务输入不消失。
- 其他块内容只读时，要提示回原文档编辑。

### 6.11 quickbar

文档内 quickbar 在点击任务块后弹出：

- 字段：状态、重要性、开始日期、截止日期、时长、备注。
- 动作：AI 优化标题、提醒、更多。
- 备注支持 Markdown 工具：无序列表、有序列表、粗体、斜体、行内代码、链接、引用。
- 时长支持预设选项和自定义输入。
- 任务行末尾可常驻显示状态、日期等字段，移动端可单独控制。

验收：

- 修改字段后通知主插件页局部刷新。
- 链接文字被选中后，点击链接按钮自动填充 Markdown 链接语法。

### 6.12 设置中心

设置页签：

- 文档分组：添加笔记本、自定义分组、文档 ID、包含子文档、排除文档、滴答 CSV 导入、导出当前分组。
- 常规设置：视图与布局、搜索与分组、顶栏入口、quickbar、番茄钟联动、数据导入、新建任务目标、状态选项、自定义字段。
- 外观：主题预设、TweakCN/shadcn CSS 变量导入、颜色、列设置、图标样式。
- 日历：初始视图、周起始日、月视图最少日程、提醒、侧栏、节假日、农历、颜色、时间范围。
- AI：启用开关、供应商 MiniMax/DeepSeek、API Key、Base URL、模型、温度、最大 Token、超时、默认上下文、排期时间段、测试连接。
- 规则管理：筛选规则列表、新建/编辑/删除。
- 四象限：按重要性和时间范围配置分组规则及颜色。
- 优先级算法：权重、规则、文档/分组加减分。
- 关于：插件环境和诊断信息。

验收：

- 设置项变更后保存到插件存储，并同步 localStorage 快捷配置。
- 切换设置项时，长列表位置不应无故跳回顶部。

### 6.13 AI 能力

AI 为可选能力：

- 可隐藏所有 AI 入口。
- 支持 MiniMax 和 DeepSeek。
- 可获取任务快照、文档快照、当前视图任务、当前分组任务。
- 可应用任务字段 patch。
- 可创建任务和子任务。
- 用于任务命名优化、自然语言字段编辑、SMART 分析和排期。

验收：

- AI 未启用或无 API Key 时，入口隐藏或给出明确配置提示。
- AI 应用任务变更后必须做结果校验或刷新确认。

## 7. 非功能需求

- 性能：大任务量场景采用缓存、增量刷新、虚拟/分批渲染，避免全量重绘。
- 兼容：支持思源 3.6.4 及以上，自定义任务复选框格式。
- 响应式：桌面、Dock、移动端、横屏均可用，控件不遮挡内容。
- 可访问性：图标按钮需有 tooltip 或 aria-label。
- 可配置：视图、字段、颜色、规则、列、入口、联动均应可配置。
- 稳定性：编辑中弹窗、详情页、悬浮条、拖拽操作需要避免被后台保存/刷新打断。

## 8. 关键验收清单

- 首次打开无配置时能引导完成文档分组配置。
- 同一任务在清单、表格、看板、日历、白板、quickbar 中字段状态一致。
- 完成任务时记录完成时间，并可被规则筛选和排序使用。
- 拖拽任务调位置、拖到日历改日期、拖到重要性图标改重要性均可用。
- 移动端短按/长按入口、详情抽屉、日历长按、新建任务流程可用。
- 自定义状态、列、字段、规则修改后，各视图同步生效。
- 附件管理支持图片、PDF、普通文件、文档链接、块链接。
- 手动刷新、切换分组、保存设置不丢失折叠状态和当前上下文。
